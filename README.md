# Holdings-lab | Policy-to-Asset Learning System

금융 정보는 많지만, 초보 투자자에게 어려운 지점은 늘 같습니다.
"이 뉴스가 내 자산에 어떤 영향을 주는가?"

`Holdings-lab`은 정책/거시 이벤트와 대표 자산(지수·ETF)의 연결을 구조적으로 보여줘,
사용자가 시장의 원인-결과를 학습할 수 있도록 돕는 금융 학습 프로젝트입니다.

## What We Build

이 프로젝트는 종목 추천 앱이 아니라 **학습 중심 금융 인텔리전스 도구**입니다.
사용자는 복잡한 원문을 직접 해석하지 않고, 아래 3가지를 중심으로 시장을 이해합니다.

- 방향성 예측: 이벤트 이후 자산 상승/하락 확률
- 변동성 급증 예측: 큰 흔들림(리스크) 발생 가능성
- 자산 민감도 분석: 어떤 이벤트에 어떤 자산이 더 민감한지

## Why This Approach

- 정책 이벤트는 일반 뉴스보다 시점/수치가 명확해 데이터화에 유리
- 대표 자산(ETF/지수)은 개별 종목보다 구조적 흐름 학습에 적합
- 목표는 "정답 제시"가 아니라 "이해 가능한 학습 피드백 제공"

## Model Strategy

- LLM: 뉴스/정책문을 JSON 기반 피처로 구조화하는 역할
- ML (XGBoost/LightGBM): 정형 피처 기반 예측 모델링
- Explainability: 변수 중요도 기반으로 예측 근거를 함께 제공

즉, LLM으로 직접 주가를 찍는 방식이 아니라,
**텍스트 구조화(LLM) + 예측/설명(트리 모델)** 분업 구조를 채택합니다.

## Data & Pipeline Principles

- 데이터 소스: 공식/신뢰 가능한 소스 우선 (예: 공공 경제지표, 검증 가능한 시세 데이터)
- 수집 전략: API 우선, 필요 시 동적 페이지 대응(헤드리스 브라우저)
- 저장/서빙: 반정형 피처 확장을 고려한 DB 설계 + API 기반 결과 전달
- 평가 원칙: 정확도 단일 지표가 아닌 클래스별 F1-score 중심 검증

## Repository Direction

현재 저장소는 아래 구조를 기준으로 발전합니다.

- `crawler/`: 데이터 수집 및 전처리
- `training/`: 모델 학습/평가
- `shared/`: 공통 유틸리티 및 스키마

## Team

- 엄지용: UI/UX 및 feature 조정
- 이석호: 크롤링 및 DB 관리
- 이찬준: XGBoost 학습 및 ETF 데이터
- 함민식: 백엔드/API 구성

---

이 프로젝트의 핵심은 **투자 행동 유도**가 아니라,
**정책 이벤트를 통해 금융 지식을 학습하고 의사결정 역량을 키우는 것**입니다.
